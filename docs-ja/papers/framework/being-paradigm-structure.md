# Beingパラダイム構造

```text
Beingパラダイム
├── ⭐ Ontological Programming（存在論的プログラミング）
│   ├── 🔑 Being Class（存在クラス）
│   │   ├── 中核原則
│   │   │   ├── 存在は行動に先行する
│   │   │   ├── 証明としての存在
│   │   │   │   ├── 構築は証明
│   │   │   │   └── セマンティック変数は証明
│   │   │   ├── Beingは不変
│   │   │   ├── エラー処理より不可能性
│   │   │   └── 組成は存在依存
│   │   ├── 時間的Being
│   │   ├── 本性
│   │   ├── 主体-客体統一
│   │   └── 「自分の本性を通じてなりたい自分になる」
│   ├── 🔑 存在契約
│   ├── コンストラクタ工房理論
│   ├── セマンティック変数名
│   ├── SchemaLogger（完全実行ストーリー）
│   ├── AMD（拡張意思決定）
│   │   ├── #[Accept] 属性（不確実性の受容）
│   │   ├── 非決定状態（知らないことの智慧）
│   │   ├── コンテキスト保存（豊富なコンテキスト維持）
│   │   └── 決定透明性（determine/フォルダ構造）
└── ⭐ Metamorphic Programming（メタモルフィック・プログラミング）
    ├── フレームワーク実装
    │   ├── 🔑 #[Be] 属性（存在論的宣言）
    │   ├── #[Input] 属性（前段階からの継承）
    │   ├── #[Inject] 属性（一時的能力注入）
    │   └── Public Readonly Properties
    ├── 🔑 型駆動メタモルフォーゼ
    │   ├── Beingプロパティ（存在論的自己決定）
    │   ├── 実存的問い
    │   ├── ユニオン型運命
    │   └── 不変名原則
    ├── メタモルフォーゼパターン
    │   ├── 線形メタモルフィック連鎖
    │   ├── Fork-Joinパターン（並列アセンブリ）
    │   ├── 並列アセンブリ
    │   └── 自然進行
    ├── 自己変換
    ├── InputとBeingの区別
    └── ⭐ being life（存在し、変化し、意味を持つオブジェクト）
```

#### 凡例
- ⭐ = 中核概念（パラダイムの基盤）
- 🔑 = 必須実装概念（理解と実装に必要）
